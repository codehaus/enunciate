Introduction

The JDK 1.4 client-side library may be used to access the SOAP API for this application via the Java runtime. This library is compatible with Java, version 1.4 and later.

The library is divided into service interfaces, service implementations, and service data. The service interfaces exist for convenience in testing and for possible aspect injection. The service implementations and service data are POJOs, making them easy to understand and straightforward to use. (The services can easily be declared in a Spring beans configuration file, for example.)

As an example, suppose there existed a service interface and method for reading a purchase order. Assuming the package "com.company", a typical use of the library would like like this:

Example

//instantiate a new service with an impl // (or through dependency injection, or whatever)... com.company.PurchaseOrderService service = new com.company.impl.PurchaseOrderServiceImpl(); //make the remote call to read the purchase order... com.company.PurchaseOrder order = service.readPurchaseOrder("123456"); //handle the purchase order as needed...

Dependencies

These client-side classes have runtime dependencies on libraries external to the JVM. These libraries must be included on the classpath when consuming this client-side API.

  • XFire is used as the SOAP stack engine. It, in turn, has dependencies on other libraries. At the time of this writing, the known dependencies were the activation framework jars, jdom, woodstox, commons-codec, and commons-httpclient. However, please consult the dependency guide for the xfire version you are using. XFire and its dependencies can be downloaded from the codehaus.
  • Enunciate XFire Client Tools is a small set of classes used as tools to enable the correct and compatible remote invocations. This jar can be downloaded from the Enunciate home page.
