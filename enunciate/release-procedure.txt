1. Update the documentation to reflect any changes.
    1a. Make sure you've updated the insert sidebox on the index page.
    1b. Make sure you've updated the downloads page to reflect where the new files will be.
    1c. Make sure you've also updated the version in the executables.html page.
    1d. Make sure you're also updated the version maven plugin in the sample poms.
2. Make sure any new modules are included
    2a. Make sure the parent pom includes any new modules.
    2b. Make sure the top pom includes any new modules.
    2c. Make sure the rt pom includes any new modules.
3. for each module, starting with parent:
    3a. change the pom to the new version.
    3b. mvn install
4. svn revert -R . (to roll back the poms to the release plugin can put them back)
5. mvn release:prepare
6. mvn release:perform

(Your new version should now be deployed.)

7. Build the distribution (e.g. "ant -f build-site.xml dist")
8. mount the dist repo (e.g. "mount /home/heatonra/mnt/enunciate/dist")
9. mount the web repo (e.g. "mount /home/heatonra/mnt/enunciate/web")
10. Copy the dist files to the dist repo (e.g. "cp -v target/*.zip target/*.tar.gz /home/heatonra/mnt/enunciate/dist")
    10a. Make sure the links you have on the downloads page are correct.
11. Copy the web content to the web repo (e.g. "rsync -av --delete target/docs/ /home/heatonra/mnt/enunciate/web")
12. Make your announcements.
    12a. Codehaus news

    12d. Mailing lists.
