introduction

Enunciate is a web service deployment framework. It is not another web service stack implementation. Rather, enunciate leverages the existing web service technologies to provide a mechanism to package, to deploy, and to clearly, accurately deliver your web service API.

Enunciate's "compiled contract" philosophy reconciles the conflict between the code-first and WSDL-first convictions by enforcing a WSDL-first mindset over a code-first implementation. The pith of the "compiled contract" philosophy is that while the web service API is developed and maintained in source code, developers are forced at compile time to reconcile any ambiguities or other potential hazards in the resulting authoritative XML contract.

Currently, Enunciate supports only service endpoints written in Java that conform to the JAX-WS (JSR-181) specification. In the future, Enunciate could be expanded to support service endpoints written in other languages (e.g. .NET).

Enunciate provides an attractive set of capabilities and features, including the following:

  • Generation of fully-documented client code for multiple client platforms, including Java 5, Java 1.3+, .NET, and C++.
  • Generation of a fully-annotated XML web service contract (i.e. WSDL with associated XML-Schema documents) from documented (i.e. JavaDoc) source code.
  • Complete packaging of the API into a J2EE-compliant web archive (war) that includes full HTML documentation of the published services.
  • Customization support for namespaces, schema locations, file names, and documentation sytylesheets.
