Roadmap

Enunciate has a long and exciting road ahead. Many features need to be added and maintained. Listed here are just a few.

Future Features

  • JSON

    Publish RESTful endpoints as JSON endpoints. Possibly use Jettison.

  • ResourceBundle

    Internationlization for the messages, client library file descriptions, and documentation headers.

  • Versioning

    Support for multiple endpoints (same name, different namespace) at the same URL.

  • JAXB: Maps

    support for java.util.Map

  • JAXB: XmlTypeAdapter

    support for the @XmlTypeAdapter.

  • JAXB: import

    Support for jaxb.index and ObjectFactory class.

  • JAXB: attribute wildcard

    support for the attribute wildcard.

  • JAXB: xml-mixed

    support for the xml-mixed content.

Future Modules

  • .NET

    Generate C# client code for accessing the API.

  • C/C++

    Generate C/C++ client code for accessing the API. See gSOAP.

  • AJAX

    Generate AJAX/JavaScript code for accessing the RESTful endpoints.

  • command

    Issue commands, scripts at specified steps in the process

  • WS-*

    Direct support for WS-Addressing, WS-ReliableMessaging, WS-Coordination, WS-Atomic Transaction, WS-Security.

  • ESB

    Direct support for integration with an enterprise service bus.

  • Standalone

    Support for a standalone program to publish the endpoints (embed jetty, tomcat or something).

  • Axis 2

    Support for axis 2 in addition to XFire (perhaps in conjuction with WS-*) and do a performance analysis between the two.

  • SMTP

    Support for an SMTP transport.

  • Jabber/IM Transport

    Support for a Jabber/IM transport.

  • HTML form

    HTML test form (like .NET).

  • Messaging

    Support for a messaging transport.

  • Relax NG

    Describe the API in Relax NG.
